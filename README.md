# astrbot_plugin_r_parser

一个面向 B 站、抖音、快手、微博、小红书链接解析插件，支持自动识别消息中的链接并提取媒体内容（视频 / 图片 / 图文 / 动态视频等）。

---

## 功能特性

- 自动识别并解析多平台链接
- 支持短链跳转与多种链接格式
- 多媒体下载与缓存
- 超限保护（体积限制）
- OneBot V11 贴表情“已受理”提示（可选）
- 抖音特殊图集（多段动态视频）增强解析
- B站评论区截图卡片（含过滤机制，可选）
- B站空间主页信息卡片渲染（代表作卡片）
- B站直播间链接卡片渲染（直播封面、主播、分区、人气等）
- 多平台下载失败自动兜底（curl_cffi / aiohttp / yt-dlp）

---

## 目前重点支持

### **Bilibili（B站）**

- BV / AV / 短链 / 动态 / opus
- 个人空间链接（`space.bilibili.com` / `m.bilibili.com/space/...`）
- 直播链接（`live.bilibili.com/...`）
- 视频下载、分P支持
- 评论区图片卡片生成
- 评论内容过滤（文本引流 + 二维码图片，视配置/依赖）
- 空间卡片代表作展示（优先代表作位，失败时兜底策略）

### **Douyin（抖音）**

- v.douyin / jx.douyin 短链
- video / note / slides 等类型
- 图集动态视频提取（`images[].video`）
- 动态视频去重与唯一命名，避免覆盖
- 多段动态视频合并 + 背景音乐合成

## 配置说明

- `performance.max_concurrent_downloads`：最大并发下载数（建议 6~9）
- `performance.source_max_size`：媒体最大体积 MB
- `performance.bili_cache_ttl`：B站信息缓存时长（秒）
- `performance.bili_space_cache_ttl`：B站空间卡片缓存时长（秒）
- `cookies.douyin_ck`：抖音 Cookie（推荐）
- `cookies.bili_ck`：B站 Cookie（推荐，能提升接口稳定性）

---

## 工作流程简述

1. 监听消息文本 / JSON 卡片
2. 匹配平台链接并路由到对应 Parser
3. 解析结构化数据并创建 `ParseResult`
4. 下载媒体（并发 + 限流 + 缓存）
5. 对部分内容进行无头渲染（评论卡片 / 空间卡片 / 直播卡片）
6. 发送消息（必要时转码或降级）

## 常见问题

### 1) 下载了多个视频，但只发出一个

通常是发送端平台限制（一次消息对多视频兼容差）。

### 2) B站评论区出现广告 / 二维码

可启用评论过滤；二维码过滤需要 `opencv-python` 和 `numpy`。

### 3) B站空间代表作偶发抓不到

可能是接口频控（如 `-799`）或目标页面结构变化。  
当前已提供缓存与兜底策略，建议配置 `bili_ck` 提升稳定性。

### 4) B站直播详情返回 `-352`

属于风控/访问校验场景。  
插件已支持多链路降级（主接口 -> fallback -> 页面信息兜底），建议配置 `bili_ck`。

### 5) ffmpeg 报错

请确认系统可执行 `ffmpeg -version`。
