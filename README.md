# astrbot_plugin_r_parser

一个面向B站、抖音、快手、微博、小红书链接解析插件，支持自动识别消息中的链接并提取媒体内容（视频 / 图片 / 图文 / 动态视频等）

---

## 功能特性

- 自动识别并解析多平台链接
- 支持短链跳转与多种链接格式
- 多媒体下载与缓存
- 超限保护（体积限制）
- OneBot V11 贴表情“已受理”提示（可选）
- 抖音特殊图集（多段动态视频）增强解析
- B站评论区截图卡片（含过滤机制，可选）
- 多平台下载失败自动兜底（curl_cffi / aiohttp / yt-dlp）

---

## 目前重点支持

- **Bilibili（B站）**
  - BV / AV / 短链 / 动态 / opus
  - 视频下载、分P支持
  - 评论区图片卡片生成
  - 评论内容过滤（文本引流 + 二维码图片，视配置/依赖）

- **Douyin（抖音）**
  - v.douyin / jx.douyin 短链
  - video / note / slides 等类型
  - 图集动态视频提取（`images[].video`）
  - 动态视频去重与唯一命名，避免覆盖
  - 多段动态视频合并 + 背景音乐合成

## 配置说明

- `performance.max_concurrent_downloads`：最大并发下载数（建议 6~9）
- `performance.source_max_size`：媒体最大体积 MB
- `cookies.douyin_ck`：抖音 Cookie（推荐）
- `cookies.bili_ck`：B站 Cookie（推荐）

---

## 工作流程简述

1. 监听消息文本/JSON卡片
2. 匹配平台链接并路由到对应 Parser
3. 解析结构化数据并创建 `ParseResult`
4. 下载媒体（并发 + 限流 + 缓存）
5. 发送消息（必要时转码或降级）

## 常见问题

### 1) 下载了多个视频，但只发出一个
通常是发送端平台限制（一次消息对多视频兼容差）。  
建议使用转发消息（Nodes）或逐条发送策略。

### 2) B站评论区出现广告/二维码
可启用评论过滤；二维码过滤需要 `opencv-python` 和 `numpy`。

### 3) ffmpeg 报错
请确认系统可直接执行 `ffmpeg -version`。
