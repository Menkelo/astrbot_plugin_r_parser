# [v1.3.0] - B站空间 / 直播卡片化与分层重构

* **🧩 Bilibili Space 主页卡片**
  * 新增空间链接识别与解析：
    * `space.bilibili.com/{mid}`
    * `m.bilibili.com/space/{mid}`
  * 生成 UP 主空间信息图（头像、昵称、签名、关注/粉丝/稿件、代表作）
  * 代表作策略优化：优先代表作位，失败时回退播放量兜底路径
  * 代表作空缺时支持隐藏作品模块，避免占位卡片影响观感

* **📡 Bilibili Live 直播卡片**
  * 新增直播链接识别与解析：
    * `live.bilibili.com/{room_id}`
  * 生成直播信息卡片（封面、标题、主播、分区、人气、状态）
  * 直播详情请求增加多链路降级与异常兜底
  * 渲染样式优化：封面比例自适应，避免裁切失真
  * 卡片增加版权信息展示（左下角）

* **🏗️ 结构分层重构**
  * B站解析进一步分层：
    * `space_service.py`（空间解析）
    * `live_service.py`（直播解析）
    * `space_renderer.py`（空间卡片渲染）
    * `live_renderer.py`（直播卡片渲染）
  * `bilibili/__init__.py` 职责收敛为路由与编排

* **🖼️ 渲染体验优化**
  * 评论区图片渲染调整为完整显示（避免 `cover` 裁切）
  * 空间卡片与直播卡片封面比例适配增强（竖图 / 超宽图）
  * 版权信息位置与视觉细节优化

* **🛠️ 稳定性与可观测性**
  * 增加链路日志可观测性（接口 code/message）
  * 频控场景增强退避与兜底策略
  * 缓存命中与回退路径优化，减少空卡输出

# [v1.2.0] - B站 / 抖音解析能力强化与结构优化

* **🚀 Bilibili 解析提速**
  * 新增视频信息与播放地址轻量缓存（`get_info / get_download_url`）
  * 减少重复请求带来的等待时间，重复解析同链接响应更快
  * 评论卡片渲染改为异步任务，避免阻塞主视频主链路

* **🧹 Bilibili 评论区质量过滤（双层）**
  * 第一层：文本规则过滤（引流词、联系方式、短链等）
  * 第二层：二维码图片识别过滤（依赖 OpenCV + numpy）
  * 支持二维码检测次数上限与超时控制，兼顾效果与性能

* **🧩 Bilibili 结构分层**
  * 拆分评论相关能力至独立模块：
    * `comment_service.py`
    * `comment_renderer.py`
    * `stream_selector.py`
  * 主解析器 `__init__.py` 聚焦路由与流程编排，可维护性提升

* **🎬 Douyin 多段动态视频解析增强**
  * 支持短链场景下的多段动态图（`images[].video`）提取
  * 修复仅识别单条媒体的问题
  * 动态视频去重逻辑增强（`uri / video_id / fallback key`）

* **📦 Douyin 下载稳定性修复**
  * 修复动态视频文件同名覆盖（如 `play.mp4`）导致丢失/重复的问题
  * 为动态视频分段下载引入唯一命名策略
  * 图集含动态视频时可按策略优先发送视频内容

* **🎞️ Douyin 合并能力**
  * 支持多段动态图自动合并为单视频
  * 支持提取背景音乐并尝试合成到合并后视频
  * 兼容无音乐场景自动回退无音轨合并结果

* **🛠️ 下载兜底与健壮性**
  * 下载链路保持多级兜底：`curl_cffi -> aiohttp -> yt-dlp`
  * 强化异常处理与日志可观测性，便于排障
